# Workflow Boundaries

## This skill is responsible for

- capturing a new task from free-form user input
- enriching an existing task
- listing or filtering backlog tasks
- updating status, priority, tags, due dates, or assignee fields
- reprioritizing one or more tasks after analysis

## This skill is not responsible for

- creating a graph workspace from a task subset
- laying out nodes or edges in a planning view
- analyzing ready or blocked work inside a specific planning graph
- opening or closing the browser workspace

## Boundary check

Stay in this skill if the user is still talking about backlog items as a list.

Switch to `roadmap-planning-views` once the user wants to turn a subset of tasks into a focused graph workspace with explicit relationships.
